Deploy anywhere

A 7.5MB binary with PostgreSQL. Runs on a Raspberry Pi or your cloud. No cluster required.

workflow.yaml 
activities:
  - key: analyze
    activity_name: llm_prompt
    parameters:
      model:
        - anthropic/claude-sonnet-4-5
        - openai/gpt-4o-mini  # fallback
      prompt: "Analyze this document..."
      max_tokens: 500
    settings:
      budget:
        limit_usd: 0.50
        action: abort  # hard limit
      cache:
        enabled: true
        ttl_seconds: 3600

Running in under 5 minutes

1

Clone and start

Run ./docker up --examples to launch with PostgreSQL.

2

Get an access token

Use the generated client credentials to authenticate via OAuth2.

3

Deploy a workflow

POST a YAML definition to /api/v1/workflow_definitions

4

Track your costs

Query /api/v1/workflows/{id}/cost for token usage.

terminal 
# Clone and start
$ git clone https://github.com/kruxia/kruxiaflow.git
$ cd kruxiaflow
$ ./docker up --examples

# Get access token
$ CLIENT_SECRET=$(grep KRUXIAFLOW_CLIENT_SECRET .env | cut -d= -f2)
$ TOKEN=$(curl -s -X POST http://localhost:8080/api/v1/oauth/token \
    -d "grant_type=client_credentials" \
    -d "client_id=kruxiaflow-docker-client" \
    -d "client_secret=$CLIENT_SECRET" | jq -r '.access_token')

# Deploy and run a workflow
$ curl -X POST http://localhost:8080/api/v1/workflow_definitions \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: text/yaml" \
    --data-binary @examples/04-moderate-content.yaml
